# 设计语言全景 · The QUENCH Design Language

> 在谈迁移、谈代码之前，先认识这门语言。这一章可独立阅读。

## 一句话

**QUENCH（淬钢）是一门为 AI / LLMOps 控制台而生的「克制智能冷钢」设计语言。**

服务对象：长时间盯屏、在模型吞吐 / 账号健康 / 营收 / 告警间做经营决策的运营者。它同时满足两个要求——

- **专业控制台的克制**：低明度深色护眼、信息密度高、不炫技、不疲劳
- **AI 行业的智能调性**：不死灰、有专业的智能感

> 关键平衡：QUENCH 的智能感来自 **颜色的语义编码**（克制、有信息价值），而**不是**辉光、渐变、发光特效（花哨、易疲劳）。**克制本身，就是 AI 工具的高级感。**

## 美学根基 · 四种物料

四种物料，每种附 **∵ 为什么**。注意：四种里没有"光效"——那是刻意的。

### 一 · 蓝紫冷石墨 —— 沉静而非死灰的基底
99% 是冷石墨，页面底 `#0A0B12` → 卡片 `#12131C` → 浮层 `#1A1C28`，带一缕**蓝紫色相（hue≈235）**，不是中性灰。

> **∵ 为什么带蓝紫**：纯中性灰在低明度下死气沉沉、像关机屏幕。微量蓝紫唤起「深空 / 算力」的智能联想，且冷色后退不抢数据。v1 的失误是把蓝紫抽干成工业灰。
> **∵ 为什么不用纯黑**：纯黑与内容明度落差过大、边缘眩光、刺眼廉价。`#0A0B12` 保留深邃又给眼睛缓冲。

### 二 · 智能三色 —— 编码语义的纯色，不发光
强调色是三个**纯色**，每色绑定"谁在动"：

| 色 | 值 | 语义 | ∵ 为什么是这个色 |
|---|---|---|---|
| **azure 蓝** | `#5CA8FF` | **人在操作**：焦点、选中、交互 | 蓝是「可点、可控」的通用 UI 语言 |
| **violet 紫** | `#8B7CFF` | **AI 在工作**：模型、推理、生成 | 紫是神经网络 / 智能的行业符号 |
| **cyan 青** | `#3DD4D0` | **数据在流动**：实时、吞吐、流量 | 青绿是数据流 / 能量的通感色 |

> **∵ 为什么三色而非一色**：单色只能分"重要/不重要"。三色让颜色**承载语义**——瞥一眼就知「蓝是我点的、紫是模型干的、青是实时数据」。颜色从装饰升级为**信息编码**。
> **∵ 为什么是纯色、不发光**：语义编码靠"色相可辨"就够了，辉光不增加任何信息，只增加视觉噪音和疲劳。**克制原则：能用纯色表达的，绝不加光。**
> **∵ 仍然克制**：三色加起来 <3%，是冷钢底上的"信号点"，不是涂料。泛滥则编码失效。

### 三 · 钢白金额 —— 全站最亮（靠亮度，不靠光晕）
金额用 `#F4F7FD` 钢白——全站最亮，配 mono + `tabular-nums`。**不加光晕。**

> **∵ 为什么钱最亮**：控制台终极目的是经营决策，钱是最高优先级信息。视觉层级=信息优先级，最重要的给最高亮度。
> **∵ 为什么不加光晕**：金额靠"比周围亮 + 等宽 mono"已足够跳出，光晕是多余的装饰。**克制：亮度够了就停手。**
> **∵ 为什么 mono + tabular-nums**：等宽数字逐位纵向对齐，扫一眼比较数量级；mono 自带精确、可信气质。比例字体数字会跳动错位。

### 四 · 锻面金属 —— 克制的质感签名
卡片、按钮用极轻的金属面：`180deg` 微渐变 + 顶部 1px `inset` 高光。**仅此而已。**

> **∵ 为什么金属而非纯扁平**：一道顶部高光（光打在钢棱）给"可触实体感"，呼应"淬钢"品牌。
> **∵ 为什么极克制**：不做厚投影、不做斜面浮雕、不做发光。只一道 1px 高光 + 微渐变——专业仪器的质感，不是消费品的塑料炫光。

## 设计语言的五种构成

### 一 · 色彩语言 —— 三层角色 + 克制编码
- **结构色**（蓝紫冷石墨 `--bg-*` / 冷白 `--ink-*` / 冷灰线 `--line-*`）：99%
- **智能三色**（`--azure` / `--violet` / `--cyan`）：编码"人/AI/数据"，纯色，<3%
- **语义色**（`--ok` / `--warn` / `--bad`）：只表状态，<1%
> 看到一处颜色，先问它属于哪种角色。答不上来，就是错的。

### 二 · 质感语言 —— 克制金属，无光效
极轻金属面。动效只服务「状态变化」与「信息层次」，**无装饰性辉光、无渐变动画**。

### 三 · 空间语言 —— 三层分治 + 密度节奏
页头 / 工具栏 / 内容区；核心上浮、细节折叠、全貌进抽屉；密度可切（44/32px）。

### 四 · 交互语言 —— 五动词 + 渐进披露
SavedViewTabs / AdvancedFilter / BulkBar / Drawer / CommandPalette + 行内3操作→弹窗→抽屉。

### 五 · 状态语言 —— 三态可视化
点 / 徽章 / 进度，择一不混。正常态用智能三色，越界染语义色。

## 设计语言的灵魂 · 八条法则

1. **克制简约** — 能不加的就不加，这是最高法则 ⭐
2. **冷峻基底** — 三色角色分治，99% 蓝紫冷钢（脱灰，不死灰）
3. **智能编码** — 蓝=人 / 紫=AI / 青=数据，纯色承载语义，不发光
4. **金额为王** — 钱是全站最亮，靠亮度不靠光晕
5. **三层分治** — 页头 / 工具栏 / 内容区
6. **渐进披露** — 行内3操作 → 弹窗 → 抽屉
7. **五动词** — 交互只用这五个，不造新词
8. **状态三态** — 点 / 徽章 / 进度，择一不混

> 八条之上还有一句：**克制不是少做，是只做对的。** 每加一个元素前问：它增加了信息，还是只增加了噪音？

# Part 1 · 视觉层规范（每条带「为什么」）

> 最精确的一章。每个数值附设计理念。暗色第一公民，亮色降级。

## 1.1 色彩系统

### 1.1.1 基底 · 蓝紫冷石墨
| Token | 暗色 | 亮色 | 用途 |
|---|---|---|---|
| `--bg-0` | `#0A0B12` | `#F4F5FA` | 页面底 |
| `--bg-1` | `#12131C` | `#FFFFFF` | 卡片 |
| `--bg-2` | `#1A1C28` | `#ECEEF5` | 浮层 / hover |
| `--bg-3` | `#232636` | `#E2E5EF` | 高浮层 / 选中底 |
| `--line-0` | `#252839` | `#DCDFEA` | 分隔线 |
| `--line-1` | `#363B52` | `#C4C9DA` | 强边框 |

> **∵ 四档背景递进**：每层明度递增 4–6%，形成「页面→卡片→浮层→选中」纵深。**层级用明度差而非边框**——∵ 深色 UI 里明度差比硬边框更柔和高级，符合"光照建立层次"的直觉。
> **∵ 基底带蓝紫**：脱灰 + 智能感 + 冷色后退。
> **∵ line 也带蓝紫**：同色相的线融入背景不打架，避免脏色。

### 1.1.2 墨色 · 冷白
| Token | 暗色 | 亮色 | 层级 | 对比(over bg-0) |
|---|---|---|---|---|
| `--ink-0` | `#E9EBF4` | `#1A1D26` | 主文字 | ~14:1 |
| `--ink-1` | `#9BA3B8` | `#4A5266` | 次级 | ~6:1 |
| `--ink-2` | `#767E96` | `#8B95AC` | 弱化 | ~4.6:1 (AA) |

> **∵ 三档墨色=三级信息权重**：用明度而非字号分层——∵ 控制台信息密度高，字号一膨胀就挤爆，靠明度分层省空间。
> **∵ 墨色带微蓝**：纯白在蓝紫底上偏冷刺眼，带一丝蓝与背景同调更柔和。
> **∵ ink-2 卡 4.6:1**：WCAG AA 正文最低线之上的安全值。弱化≠看不清。

### 1.1.3 智能三色 · 纯色语义编码
| Token | 暗色 | 亮色 | 语义 | 淡背景 |
|---|---|---|---|---|
| `--azure` | `#5CA8FF` | `#3D91F0` | 人/焦点/交互/选中 | `--azure-dim .14` |
| `--violet` | `#8B7CFF` | `#6E5DE0` | AI/模型/推理/生成 | `--violet-dim .14` |
| `--cyan` | `#3DD4D0` | `#1BA9A4` | 数据/实时/吞吐 | `--cyan-dim .14` |

**用法边界（铁律）**：
- `azure` → 输入焦点、选中行、激活页签、主按钮、可点链接
- `violet` → 模型名标签、AI 推理状态、"生成中"、智能推荐、AI 类 KPI
- `cyan` → 实时 RPM/TPM、心跳、吞吐曲线、数据流动

> **∵ 严格绑定语义**：三色价值全在"可预测"。violet 被挪去做装饰，"紫=AI"编码就崩。纪律是编码前提。
> **∵ 纯色、不发光**：色相可辨即达成编码，辉光只增噪音与疲劳。**克制：能用纯色就不加光。**
> **∵ dim 统一 .14**：太淡看不出、太浓抢前景，.14 是"感知到色相但不喧宾"的甜点。
> **∵ 三色明度拉平**（都 L≈70）：它们是平级活性指示，无谁更亮的层级暗示——层级交给金额和墨色。

### 1.1.4 焦点指示（功能性，非装饰）
| Token | 值 | 用途 |
|---|---|---|
| `--ring-focus` | `0 0 0 1.5px var(--azure)` | 输入/行/键盘焦点环 |

> **∵ 为什么保留焦点环**：焦点可见是 **a11y 硬性要求**（键盘导航必须看得见焦点在哪）——这是功能，不是装饰，不能砍。
> **∵ 为什么只用 1.5px 实环、不加外发光**：实环精确指示边界已够用。v2 那层 20px 外发光晕是装饰性的，砍掉。**克制：焦点要"可见"，不要"发光"。**

### 1.1.5 钢白金额
| Token | 暗色 | 亮色 |
|---|---|---|
| `--money` | `#F4F7FD` | `#0F1320` |

工具类 `.q-money` = mono + `tabular-nums` + `--money`。**无 text-shadow 光晕。**
> **∵** 金额靠"最亮 + 等宽"已足够跳出，光晕是多余装饰，砍。亮色下用近黑（最重要靠最深），逻辑对称。

### 1.1.6 金属面（极克制）
| Token | 暗色值 |
|---|---|
| `--metal` | `linear-gradient(180deg, #181A24, #101119)` |
| `--metal-raised` | `linear-gradient(180deg, #2A2E40, #161823)` |
| `--edge-hi` | `inset 0 1px 0 rgba(255,255,255,.06)` |

> **∵ 渐变 180°（上→下）**：模拟顶部光源，上亮下暗显"凸起可按"。
> **∵ raised 更亮**：用于按钮等"要凸出被点"的元素，明度高=靠前。
> **∵ 渐变幅度极小**：metal 上下色差仅约 8 明度——∵ 只要一丝质感暗示即可，大幅渐变就花哨了。

### 1.1.7 语义色 · 状态专用
| Token | 暗色 | 亮色 | 含义 | 淡背景 |
|---|---|---|---|---|
| `--ok` | `#4FD89A` | `#1E9E65` | 成功/活跃/健康 | `--ok-dim` |
| `--warn` | `#F0BE54` | `#C48A1A` | 警告/限流/临界 | `--warn-dim` |
| `--bad` | `#FF6470` | `#D93F4C` | 危险/错误/离线 | `--bad-dim` |
| `--info` | `#5CA8FF`(=azure) | `#3D91F0` | 中性提示 | `--info-dim` |

> **∵ v2 提亮提饱和**（v1 ok 偏暗 #46C98C）：状态色是告警信号，在低明度底上必须够亮才能第一时间抓眼；提饱和也给画面注入活力，对治灰暗。
> **∵ 绝不装饰**：语义色用来装饰，用户就会对"绿=好"脱敏，真告警被淹没。信号必须稀有才有效。
> **∵ info 复用 azure**：中性提示与"可交互"气质一致，无需再造色。

### 1.1.8 冻结区（禁用）
`src/style.css:13-20` 的 shadcn HSL 变量待淘汰。新代码禁用 `bg-background` `text-foreground` `text-primary-N` `bg-{gray,slate,zinc}-N`。

## 1.2 排版系统

### 1.2.1 字体族
| 用途 | 栈 | ∵ |
|---|---|---|
| 正文 | `Archivo → Inter → system-ui → 'PingFang SC'` | 几何无衬线，中性现代、可读性高、不带情绪——让数据说话 |
| 数字 | `'IBM Plex Mono' → ui-monospace → SFMono → Menlo` | 等宽逐位对齐可比；自带工程精确气质 |
> **∵ 不用衬线/圆体**：衬线太人文、圆体太消费，都不符精密控制台调性。几何无衬线是工具类共识。

### 1.2.2 字号阶梯
| 角色 | 字号 | 字重 | ∵ |
|---|---|---|---|
| 页面标题 | 21px | 700 | 唯一大字，建立页面身份 |
| 区块标题 | 14px | 600 | 靠字重而非字号分层 |
| 正文/表格 | 13px | 400–500 | 密度与可读最优 |
| 辅助/标签 | 12px | 400 | 次要信息 |
| 微标/角标 | 10–11px | 600 | 徽章状态标，字距加宽 |
> **∵ 阶梯克制（21/14/13/12）**：控制台单屏塞大量信息，字号一大就挤爆。QUENCH 用**字重+明度**建层级，把字号差压到最小——这是信息密集界面区别于营销页的根本排版策略。

### 1.2.3 数字排版
金额/统计/计数：`tabular-nums` 等宽，保证纵向可比。

## 1.3 间距系统

### 1.3.1 8pt 网格
所有间距取 4 的倍数（4/8/12/16/20/24/28/32/48/64）。
> **∵ 为什么用网格**：统一步长给界面"看不见的秩序"——精致感的来源。随意的 7/13/19px 间距让界面显脏、乱、业余。

### 1.3.2 关键间距
| 场景 | 值 | ∵ |
|---|---|---|
| 页面 padding | `24px 28px 64px` | 顶 24 给页头呼吸；底 64 留滚动余量；左右 28 不贴边不浪费 |
| 强关联间距 | `12px` | 相关元素（标题+描述）小间距表"一组" |
| 分隔间距 | `16–20px` | 独立区块大间距表"另一块" |
| 卡片内 padding | `16–24px` | 密集卡 16，主卡 24 |
| 表格行高 | `44px / 32px` | comfortable / compact |
> **∵ 间距即语义**：间距小=关系近、大=关系远（格式塔邻近原则）。用间距编码"亲疏"，不只是美观。

### 1.3.3 圆角
| 值 | 用途 | ∵ |
|---|---|---|
| `--q-radius: 12px` | 卡片 | 太方=工业冷硬，太圆=消费软萌，12px 是精密仪器的圆角 |
| `10px` | 输入/按钮 | 比卡片略小，圆角与元件尺寸成比例 |
| `6–8px` | 徽章/小标 | 更小元件更小圆角 |
| `20px` | 登录卡 | 门面亲和，唯一允许跳出冷峻的场景 |

## 1.4 阴影与高度
| 项 | 值 | ∵ |
|---|---|---|
| `--edge-hi` | `inset 0 1px 0 rgba(255,255,255,.06)` | 锻面顶高光，给"可触"暗示 |
| 卡片阴影 | `0 1px 2px rgba(0,0,0,.5), 0 4px 14px rgba(0,0,0,.4)` | 双层：近实影+远柔影 |
> **∵ 深色 UI 慎用阴影**：深色上黑影几乎看不见。QUENCH 主要靠**明度分层（bg-0→3）**建高度，阴影只辅助。
> **∵ 焦点用环不用光晕**：见 1.1.4，可见即可，不发光。

## 1.5 动效
| 动效 | 用途 | ∵ |
|---|---|---|
| `rise` (+delay .02/.06/.10s) | 入场逐层浮起 | 引导视线按优先级落点，非一次性糊脸 |
| `breath` (2.4s) | 实时心跳点 | 缓慢呼吸=活着但平静 |
| `pulse` | 告警脉冲 | 急促=需注意，与 breath 对比编码紧急度 |
| `shimmer` | 骨架屏 | 微光扫过=加载中，比转圈高级 |
> **∵ 动效总纲**：只编码「状态变化」与「信息层次」，**绝不为炫技而动**。无意义动效是噪音。
> **∵ 砍掉 metal-sheen 扫光**：hover 金属扫光是纯装饰，不增信息，砍。**克制。**
> **∵ 时长 ≤ .3s**：控制台高频操作，动效慢会拖累节奏。

## 1.6 暗 / 亮色机制
```css
:root.dark, .dark { /* 暗色：第一公民 */ }
:root           { /* 亮色：降级 */ }
```
> **∵ 暗色优先**：长时间盯屏，暗色护眼、降疲劳、契合 AI 调性。亮色是降级适配。组件只查 CSS 变量，无需条件渲染。
